## 1. الوصف العام

شاشة الشركة هي **واجهة ويب** تُعرض على شاشة/تلفاز في مدخل الشركة، تسمح للمشرف (Admin) بـ:

1. **تسجيل الدخول** مرة واحدة (JWT دائم — 10 سنوات)
2. **توليد رمز QR** لتسجيل الحضور أو الانصراف
3. **عرض QR** مع عداد تنازلي للثواني المتبقية + عرض "مدة الصلاحية"
4. **اكتشاف تلقائي** عند مسح الموظف للرمز — تختفي الشاشة فوراً
5. **إظهار حالة الفشل** إذا منع النظام التسجيل (موظف مسجل حضور، غرض خاطئ، إلخ)
6. **العودة التلقائية** للشاشة الرئيسية بعد الاستخدام أو انتهاء الصلاحية

## 3. تسجيل الدخول

### الشاشة
صفحة بسيطة تطلب **البريد الإلكتروني** و **كلمة المرور** لحساب المشرف.

### آلية التخزين
```javascript
// يتم تخزين JWT بشكل دائم في localStorage
localStorage.setItem('company_token', TOKEN);
```

| الخاصية | القيمة |
|---------|--------|
| **مدة الجلسة** | دائمة (10 سنوات) — نفس صلاحية توكن EMPLOYEE |
| **التخزين** | `localStorage` (يظل حتى مسح يدوي) |
| **استعادة الجلسة** | عند فتح الصفحة، يتحقق من `localStorage` ويعيد التوجيه تلقائياً |
| **تسجيل الخروج** | زر في أعلى يسار الشاشة |

### التحقق التلقائي من الجلسة
عند فتح الصفحة:
1. يتحقق من وجود `company_token` في `localStorage`
2. يستدعي `GET /auth/profile` للتأكد من صحة التوكن ودور `ADMIN`
3. إذا صالح → ينتقل مباشرة إلى Dashboard
4. إذا غير صالح → يحذف التوكن ويعرض صفحة تسجيل الدخول

## 5. شاشة QR والعد التنازلي

### عند الضغط على "تسجيل حضور" أو "تسجيل انصراف"

```
POST /admin/attendance/generate-qr
Body: { "purpose": "CHECK_IN" }
← { id, qrCode, purpose, expiresAt, expirySeconds }
```

### ماذا يحدث
1. يستدعي API توليد QR
2. **يقرأ `expirySeconds`** من الاستجابة (القادمة من قاعدة البيانات)
3. **يرسم QR** في واجهة HTML باستخدام مكتبة `qrcodejs` (CDN)
4. **يعرض الوقت المتبقي** بعداد تنازلي (expirySeconds ← 0 ثانية)
5. **يعرض "مدة الصلاحية: X ثانية"** تحت العداد
6. إذا تبقى **5 ثوانٍ أو أقل** → يتحول العداد للأحمر مع وميض

### مثال لشاشة QR
```
┌─────────────────────────────┐
│         امسح رمز QR          │
│   وجه كاميرا هاتفك نحو الشاشة│
│                             │
│      [✅ تسجيل حضور]         │
│                             │
│      ┌─────────────┐        │
│      │   ███████   │        │
│      │   ██ QR ██  │        │
│      │   ███████   │        │
│      └─────────────┘        │
│                             │
│           25                │
│       ثانية متبقية           │
│  ⏱️ مدة الصلاحية: 30 ثانية  │
│                             │
│       [العودة للرئيسية]     │
└─────────────────────────────┘
```

## 6. مدة صلاحية QR (من قاعدة البيانات)

### المصدر
تُقرأ مدة الصلاحية من جدول **`system_settings`** في قاعدة البيانات:

```sql
system_settings.setting_key = 'qr_token_expiry_seconds'
system_settings.setting_value = '30'  ← القيمة الافتراضية
```

### كيف تعمل
```
system_settings (qr_token_expiry_seconds = 30)
        ↓
AttendanceService.getQrExpirySeconds() → await this.systemSettingRepository.findOne(...)
        ↓
generateQr() → expiresAt.setSeconds(expiresAt.getSeconds() + expirySeconds)
        ↓
الاستجابة ← { id, qrCode, purpose, expiresAt, expirySeconds }
        ↓
شاشة الشركة ← تعرض "مدة الصلاحية: X ثانية"
```

### تغيير المدة
يمكن للمشرف تغيير المدة مباشرة في قاعدة البيانات (دون إعادة تشغيل):
```sql
UPDATE system_settings 
SET setting_value = '60' 
WHERE setting_key = 'qr_token_expiry_seconds';
```

## 7. آلية اكتشاف المسح (Polling)

### كيف تعمل
بعد توليد QR، تبدأ الشاشة **استدعاء دوري** كل 1 ثانية:

```
GET /admin/attendance/qr-status/:id
← { used: true/false, scanError: null | string, expiresAt, purpose }
```

### الحالات الأربع

| الحالة | الموقف | الإجراء |
|--------|--------|---------|
| **`used: true` + `scanError: null`** | موظف مسح QR ونجح التسجيل | ✅ عرض رسالة نجاح 2 ثانية ← العودة للرئيسية |
| **`used: true` + `scanError: "..."`** | موظف مسح QR ولكن فشل التسجيل | ❌ عرض رسالة خطأ 3 ثوانٍ ← العودة للرئيسية |
| **`expiresAt < now`** | مرت المدة دون مسح | ⏰ عرض رسالة "انتهت الصلاحية" ← العودة للرئيسية |
| **`used: false`** (ضمن المدة) | QR لم يستخدم بعد | استمرار العد التنازلي والـ polling |

### رسالة النجاح ✅
```
┌─────────────────────────────┐
│            ✅               │  (أخضر)
│     تم تسجيل الحضور         │
│   تم تسجيل حضور الموظف بنجاح│
└─────────────────────────────┘
  ↑ تختفي تلقائياً بعد 2 ثانية
```

### رسالة الفشل ❌
```
┌─────────────────────────────┐
│            ❌               │  (أحمر)
│        فشل التسجيل          │
│ أنت مسجل حضور حالياً، يرجى  │
│    تسجيل انصراف أولاً       │
└─────────────────────────────┘
  ↑ تختفي تلقائياً بعد 3 ثوانٍ
```

### كود الـ Polling (JavaScript)
```javascript
pollingInterval = setInterval(async () => {
  const res = await fetch(API + '/admin/attendance/qr-status/' + tokenId, {
    headers: { 'Authorization': 'Bearer ' + TOKEN }
  });
  const data = await res.json();
  if (data.used) {
    stopPolling();
    if (data.scanError) {
      showFailure(data.scanError, purpose);  // ❌ QR مستخدم ولكن فشل
    } else {
      showSuccess(purpose);                   // ✅ QR مستخدم ونجح
    }
  }
}, 1000);
```

## 8. معالجة فشل المسح (scanError)

### متى يحدث الفشل؟

| الحالة | رسالة الخطأ | يظهر في |
|--------|-------------|---------|
| موظف مسجل حضور ولم ينصرف بعد | `أنت مسجل حضور حالياً، يرجى تسجيل انصراف أولاً` | تطبيق الموظف + شاشة الشركة |
| موظف لم يسجل حضور (وانصراف) | `يجب تسجيل الحضور أولاً` | تطبيق الموظف + شاشة الشركة |
| مسح QR حضور بدلاً من انصراف | `هذا الرمز مخصص للحضور وليس الانصراف` | تطبيق الموظف + شاشة الشركة |
| مسح QR انصراف بدلاً من حضور | `هذا الرمز مخصص للانصراف وليس الحضور` | تطبيق الموظف + شاشة الشركة |

### آلية العمل
عند مسح QR، يقوم `checkIn`/`checkOut` بـ:
1. استهلاك الرمز (`used = true`)
2. إذا نجح → `scanError = null`
3. إذا فشل → `scanError = "رسالة الخطأ"` ثم `throw BadRequestException`

شاشة الشركة تقرأ `scanError` عبر الـ polling وتعرضها.

### تدفق البيانات
```
موظف يمسح QR وهو مسجل حضور
        ↓
checkIn():
  qrToken.used = true
  qrToken.scanError = 'أنت مسجل حضور حالياً...'
  save() + throw()
        ↓
تطبيق الموظف ← "أنت مسجل حضور حالياً..."
        ↓
Polling شاشة الشركة:
  GET qr-status/:id → { used: true, scanError: 'أنت مسجل حضور حالياً...' }
        ↓
شاشة الشركة ← ❌ "فشل التسجيل: أنت مسجل حضور حالياً..."
```

## 9. حالة الاستخدام (Use Cases)

### السيناريو 1: حضور موظف ✅
1. المشرف يضغط **"تسجيل حضور"**
2. QR يظهر على الشاشة مع عداد 30 ثانية
3. الموظف يمسح QR بتطبيق الجوال
4. الشاشة تكتشف `used: true` + `scanError: null` ← تظهر ✅ "تم تسجيل الحضور"
5. بعد 2 ثانية ← العودة للشاشة الرئيسية

### السيناريو 2: انصراف موظف 🚪
1. المشرف يضغط **"تسجيل انصراف"**
2. QR يظهر مع عداد 30 ثانية
3. الموظف يمسح QR
4. الشاشة تكتشف `used: true` + `scanError: null` ← تظهر 🚪 "تم تسجيل الانصراف"
5. بعد 2 ثانية ← العودة للشاشة الرئيسية

### السيناريو 3: موظف مسجل حضور يحاول الحضور مرة أخرى ❌
1. المشرف يضغط **"تسجيل حضور"**
2. QR يظهر مع عداد 30 ثانية
3. الموظف (المسجل حضور حالياً) يمسح QR
4. تطبيق الموظف ← "أنت مسجل حضور حالياً، يرجى تسجيل انصراف أولاً"
5. شاشة الشركة تكتشف `scanError` ← تظهر ❌ "فشل التسجيل: أنت مسجل حضور حالياً..."
6. بعد 3 ثوانٍ ← العودة للشاشة الرئيسية

### السيناريو 4: موظف يمسح QR لغرض خاطئ ❌
1. المشرف يضغط **"تسجيل حضور"** (QR للدخول)
2. الموظف يمسحه في شاشة الانصراف
3. تطبيق الموظف ← "هذا الرمز مخصص للانصراف وليس الحضور"
4. شاشة الشركة ← ❌ "فشل التسجيل: هذا الرمز مخصص للانصراف..."

### السيناريو 5: انتهاء صلاحية QR ⏰
1. المشرف يولد QR
2. يمر وقت المدة دون مسح
3. الشاشة تعرض ⏰ "انتهت صلاحية الرمز"
4. المشرف يضغط "العودة للرئيسية" ويولد رمز جديد

### السيناريو 6: انتهاء جلسة المشرف
1. توكن المشرف انتهى (نادر — لأنه دائم)
2. Polling يستقبل `401 Unauthorized`
3. يحذف التوكن ويعرض صفحة تسجيل الدخول

## 10. API Endpoints المستخدمة

| الطريقة | المسار | الاستخدام |
|---------|--------|-----------|
| `POST` | `/auth/login` | تسجيل دخول المشرف |
| `GET` | `/auth/profile` | التحقق من صحة التوكن ودور ADMIN |
| `POST` | `/admin/attendance/generate-qr` | توليد رمز QR |
| `GET` | `/admin/attendance/qr-status/:id` | استعلام حالة QR (مستخدم؟ خطأ؟) |

### `POST /admin/attendance/generate-qr`

**الاستجابة (201 Created):**
```json
{
  "id": 5,
  "qrCode": "4815caeb76413c5c0402ad9c5b3a8b384e1f013945a775577323e71c7b8765f5",
  "purpose": "CHECK_IN",
  "expiresAt": "2026-05-19T02:28:59.000Z",
  "expirySeconds": 30
}
```

| الحقل | النوع | الوصف |
|-------|-------|-------|
| `id` | number | معرف رمز QR (للاستعلام عن الحالة) |
| `qrCode` | string | النص المراد تحويله إلى QR |
| `purpose` | string | `CHECK_IN` أو `CHECK_OUT` |
| `expiresAt` | string | وقت انتهاء الصلاحية (UTC) |
| `expirySeconds` | number | مدة الصلاحية بالثواني (مقروءة من DB) |

### `GET /admin/attendance/qr-status/:id`

**المسار:** `GET /api/admin/attendance/qr-status/5`  
**الصلاحية:** ADMIN فقط  
**الاستجابة (200 OK):**
```json
{
  "used": true,
  "scanError": "أنت مسجل حضور حالياً، يرجى تسجيل انصراف أولاً",
  "expiresAt": "2026-05-19T02:28:59.000Z",
  "purpose": "CHECK_IN"
}
```

| الحقل | النوع | الوصف |
|-------|-------|-------|
| `used` | boolean | `true` إذا تم استخدام QR (مسح من موظف) |
| `scanError` | string/null | `null` إذا نجح التسجيل، رسالة الخطأ إذا فشل |
| `expiresAt` | string | وقت انتهاء الصلاحية (UTC) |
| `purpose` | string | `CHECK_IN` أو `CHECK_OUT` |

**الخطأ:**
```json
400 Bad Request
{ "message": "رمز QR غير موجود" }
```
